对新序列采样

训练一个序列模型之后，要想了解到这个模型学到了什么，一种非正式的方法就是进行一次新序列采样。

一个序列模型模拟了任意特定单词序列的概率，要做的就是对这些概率分布进行采样来生成一个新的单词序列。

除了基于词汇的RNN模型，还可以构建基于字符的RNN结构。

循环神经网络的梯度消失

训练很深的神经网络时，随着层数的增加，导数有可能指数型的下降或者指数型的增加，我们可能会遇到梯度消失或者梯度爆炸的问题。梯度爆炸的问题的一个解决方法就是用梯度修剪。GRU，门控循环单元网络，可以有效地解决梯度消失的问题，并且能够的神经网络捕获更长的长期依赖。

使用词嵌入

用词嵌入做迁移学习的步骤：

第一步，先从大量的文本集中学习词嵌入。

第二步，可以用这些词嵌入模型把它迁移到新的只有少量标注训练集的任务中，比如说用这个300维的词嵌入来表示你的单词。

第三步，当在新的任务上训练模型时，只有少量的标记数据集上，可以选择要不要继续微调，用新的数据调整词嵌入。实际中，只有这个第二步中有很大的数据集你才会这样做。

词嵌入的特性

词嵌入有一个特性是它能帮助实现类比推理。例如找出一个向量也就是找出一个词，使得$e_{\text{man} }-e_{\text{woman} }$≈$\ e_{\text{king} }- e_{?}$，即

$Find\ word\ w:argmax \ Sim(e_{w},e_{\text{king} } - e_{\text{man} } + e_{\text{woman} })$。

相似度可以使用余弦相似度：${CosineSimilarity(u, v)} = \frac {u . v} {||u||_2 ||v||_2} = cos(\theta) \tag{1}$。

嵌入矩阵

当应用算法来学习词嵌入时，实际上是学习一个嵌入矩阵。假设这个嵌入矩阵叫做矩阵$E$，如果用$E$去乘以右边的one-hot向量也就是，那么就会得到一个低维的词嵌入向量。

学习词嵌入

算法以某个词作为目标词，选取目标词的上下文作为输入，来预测目标词，用来学习词嵌入。

Word2Vec

在Skip-Gram模型中，要做的是抽取上下文和中心词配对，来构造一个监督学习问题，它给定中心词，要求预测在这个词正负$k$个词距内随机选择的某个上下文目标词。

预测不同目标词的概率：$Softmax:p\left( t \middle| c \right) = \frac{e^{\theta_{t}^{T}e_{c} } }{\sum_{j = 1}^{10,000}e^{\theta_{j}^{T}e_{c} } }$。

损失函数：$L\left( \hat y,y \right) = - \sum_{i = 1}^{10,000}{y_{i}\log \hat y_{i} }$。

CBOW模型，即连续词袋模型（ContinuousBag-Of-Words Model），它获得中间词两边的的上下文，然后用周围的词去预测中间的词。

负采样

选择一个上下文词，再选一个目标词，作为一个正样本，上下文，目标词，并给定标签为1。然后用相同的上下文词，再从字典中选取随机的词，标记0，这些就会成为负样本。

然后使用逻辑回归模型，得到给定输入的$c$，$t$对的条件下，$y=1$的概率：$P\left( y = 1 \middle| c,t \right) = \sigma(\theta_{t}^{T}e_{c})$。

GloVe 词向量

GloVe代表用词表示的全局变量（global vectors for word representation）。

假定$X_{ {ij} }$是单词$i$在单词$j$上下文中出现的次数，定义上下文和目标词为任意两个位置相近的单词，假设是左右各10词的距离，那么$X_{ {ij} }$就是一个能够获取单词$i$和单词$j$出现位置相近时或是彼此接近的频率的计数器。

GloVe模型做的就是进行优化，将他们之间的差距进行最小化处理：

$\text{mini}\text{mize}\sum_{i = 1}^{10,000}{\sum_{j = 1}^{10,000}{f\left( X_{ {ij} } \right)\left( \theta_{i}^{T}e_{j} + b_{i} + b_{j}^{‘} - logX_{ {ij} } \right)^{2} } }$

情感分类

可以用RNN来做情感分类。使用RNN的计算一个特征表示，用来预测$\hat y$。

词嵌入除偏

怎样辨别出与某个偏见相似的趋势呢？主要有以下三个步骤：

一、对于性别歧视这种情况来说，能做的是$e_{\text{he} }-e_{\text{she} }$，因为它们的性别不同，然后将$e_{\text{male} }-e_{\text{female} }$，然后将这些值取平均。这个趋势看起来就是性别趋势或说是偏见趋势。有的趋势与想要尝试处理的特定偏见并不相关，是个无偏见趋势。

二、中和步骤，对于那些定义不确切的词可以将其处理一下，避免偏见。

三、均衡步，意思是说可能会有这样的词对，grandmother和grandfather，或者是girl和boy，对于这些词嵌入，只希望性别是其区别。

序列模型和注意力机制

基础模型

seq2seq（sequence to sequence）模型

通过输入一个法语句子，将它翻译成一个英语句子。首先建立一个网络，这个网络叫做编码网络（encoder network），它是一个RNN的结构。每次只向该网络中输入一个法语单词，将输入序列接收完毕后，这个RNN网络会输出一个向量来代表这个输入序列。之后可以建立一个解码网络，它以编码网络的输出作为输入，之后它可以被训练为每次输出一个翻译后的单词，一直到它输出序列的结尾或者句子结尾标记，这个解码网络的工作就结束了。

选择最可能的句子

假如想真正地通过模型将法语翻译成英文，通过输入的法语句子模型将会告诉你各种英文翻译所对应的可能性。当你使用模型来进行机器翻译时，要找到一个英语句子$y$，使得条件概率$P(y|x)$最大化，最通用的算法就是束搜索(Beam Search)。

集束搜索

贪婪算法只会挑出最可能的那一个单词，然后继续。而集束搜索则会考虑多个选择，集束搜索算法会有一个参数B，叫做集束宽（beam width）。在集束宽为B时，集束搜索一次考虑B个可能结果。如果集束宽等于1，只考虑1种可能结果，这实际上就变成了贪婪搜索算法。

改进集束搜索

长度归一化（Length normalization）就是对束搜索算法稍作调整的一种方式。

束搜索是最大化这个概率，这个乘积就是$P(y^{< 1 >}\ldots y^{< T_{y} }>|X)$，可以表示成:$P(y^{<1>}|X)$ $P(y^{< 2 >}|X,y^{< 1 >})$ $P(y^{< 3 >}|X,y^{< 1 >},y^{< 2>})$…$P(y^{< T_{y} >}|X,y^{<1>},y^{<2>}\ldots y^{< T_{y} - 1 >})$

一种改进是它归一化，通过除以翻译结果的单词数量。这样就是取每个单词的概率对数值的平均了，这样很明显地减少了对输出长的结果的惩罚。

一个更柔和的方法是在$T_{y}$上加上指数$a$，它是$[0, 1]$之间的数。

则目标变为：
${\arg\max}_{y}\frac{1}{ {T_{y} }^{a} }P(y^{< T_{y} >}|X,y^{<1>},y^{<2>}\ldots y^{< T_{y} - 1 >})$

集束搜索的误差分析

模型有两个主要部分，一个是神经网络模型，或说是序列到序列模型（sequence to sequence model），将这个称作是RNN模型，它实际上是个编码器和解码器（ an encoder and a decoder）。另一部分是束搜索算法，以某个集束宽度B运行。用RNN模型和束搜索算法分别计算$y^*$和$P(\hat {y}|x)$，然后通过比较这两个哪个更大来判断错误出在哪里。
